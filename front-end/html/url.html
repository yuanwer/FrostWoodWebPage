#URL

URL（统一资源定位符）是互联网上用来确定资源位置的"地址"，就像现实世界中的门牌号一样，帮助我们准确找到需要的资源。

#URL 的标准格式

[协议类型]://[服务器地址]:[端口号]/[文件路径][文件名]?[查询]#[片段ID]

让我们通过一个实际的例子来理解：

https://john.doe@www.example.com:123/forum/questions/?tag=networking&order=newest#top

各部分详细说明：

  • scheme（协议类型）：https://
  • userinfo（用户信息）：john.doe@
  • host（主机名）：www.example.com
  • port（端口）：:123
  • path（路径）：/forum/questions/
  • query（查询参数）：?tag=networking&order=newest
  • fragment（片段标识）：#top

#URL 的基本组成

在日常使用中，一个典型的 URL 通常包含以下必要部分：

https://www.example.com:8080/path/to/resource?name=test#section1
  • 协议（Protocol）：https:// - 告诉浏览器使用什么协议
  • 主机名（Hostname）：www.example.com - 服务器的域名或 IP 地址
  • 端口（Port）：:8080 - 可选，默认 HTTP 为 80，HTTPS 为 443
  • 路径（Path）：/path/to/resource - 资源在服务器上的具体位置
  • 查询参数（Query）：?name=test - 向服务器传递的额外参数
  • 片段（Fragment）：#section1 - 页面内部的锚点

#URL 编码

URL 只能使用 ASCII 字符集来通过因特网进行发送。如果 URL 中包含中文、空格等特殊字符，就需要进行编码。

在前端开发中，我们经常会用到以下方法处理 URL 编码：

// 编码
encodeURI('https://example.com/路径'); 
// 结果：https://example.com/%E8%B7%AF%E5%BE%84

// 解码
decodeURI('https://example.com/%E8%B7%AF%E5%BE%84');
// 结果：https://example.com/路径

#在前端中的常见应用

#1. 路由导航

// React Router 示例
<Link to="/user?id=123">用户详情</Link>

// 获取查询参数
const searchParams = new URLSearchParams(window.location.search);
const userId = searchParams.get('id'); // "123"

#2. AJAX 请求

// 发起 GET 请求
fetch('https://api.example.com/users?page=1')
  .then(response => response.json())
  .then(data => console.log(data));

#URL 最佳实践

WARNING

永远不要在 URL 中包含敏感信息（如密码、token 等），因为：

  1. URL 可能会被浏览器记录
  2. URL 可能会出现在服务器日志中
  3. URL 可能会被第三方看到（如通过 Referer 头）

#常见问题解决

#1. 处理中文参数

// 正确处理中文参数
const keyword = '前端开发';
const url = `https://api.example.com/search?q=${encodeURIComponent(keyword)}`;

#2. 解析 URL 参数

const params = new URLSearchParams(window.location.search);
const page = params.get('page');
const sort = params.get('sort');

#URI 和 URL 的区别

:::info URI 和 URL 的关系 URI（统一资源标识符）和 URL（统一资源定位符）是两个经常被混淆的概念，让我们来理解它们的区别：

  • URI（Uniform Resource Identifier）：统一资源标识符，用于标识 Web 技术使用的逻辑或物理资源
  • URL（Uniform Resource Locator）：统一资源定位符，俗称网络地址，相当于网络中的门牌号

URI 在以下几个方面与 URL 有关联：

  1. URI 是一个规则下能把一个资源独一无二的识别出来
  2. URL 作为一个网络 Web 资源的地址，可以唯一将一个资源识别出来，所以 URL 是一个 URI
  3. 所以 URL 是 URI 的一个子集
  4. 但是 URI 并不一定是 URL :::

:::tip URI 和 URL 的具体示例 让我们通过具体的例子来理解 URI 和 URL 的区别：

URL 示例（同时也是 URI）：

  • https://www.example.com/index.html
  • ftp://ftp.example.com/files/
  • mailto:user@example.com

仅是 URI 而不是 URL 的示例：

  • urn:isbn:0-486-27557-4 （书籍的 ISBN 编号）
  • tel:+1-816-555-1212 （电话号码）
  • data:text/plain;charset=UTF-8,Hello%20World （数据 URI）

关键区别：

  • URL 告诉你如何访问这个资源（比如通过 https 协议访问 example.com）
  • URI 只是标识这个资源（比如一本书的 ISBN），但不一定告诉你如何访问它 :::
