#JS 原型链

JavaScript 的原型链是其最具特色的机制之一,是实现对象继承的基础,也是理解 JavaScript 面向对象编程的关键概念。

#对象的创建

要理解原型链的全貌，我们需要从一个最基本的问题开始思考："对象是从哪里来的？"

在 JavaScript 中，对象都是通过 new 操作符和构造函数创建出来的。即使是最简单的对象字面量，本质上也是通过构造函数创建的：

// 使用字面量方式创建对象
const obj = {};

// 上面的写法实际上是下面这种方式的语法糖
const obj = new Object();

明白了对象的创建方式，我们来深入理解原型链的概念。

#函数与原型对象

在 JavaScript 中，每个函数都有一个特殊的"孪生兄弟" —— 原型对象（prototype）。当我们通过 new 操作符使用函数创建实例时，这个实例会自动获得一个特殊的属性 __proto__（隐式原型），它指向创建该实例的函数的原型对象。

让我们通过一个例子来说明：

function Person() {}
const person = new Person();

console.log(person.__proto__ === Person.prototype); // true

这个图展示了函数、原型对象和实例之间的基本关系：函数通过 prototype 属性连接到原型对象，实例通过 proto 属性连接到原型对象。

#原型链的形成

那么，Person.prototype 本身又是什么呢？它其实也是一个对象，是通过 new Object() 创建的。因此，它也有自己的 __proto__，指向 Object.prototype：

特别需要注意的是，Object.prototype 原型对象，它的 __proto__ 被 js 引擎默认设置为 null。这样做是为了避免无限循环的原型链。

这个 null 就是原型链的顶端。

为什么会导致无限循环

如果 js 引擎没有设置 Object.prototype 的 __proto__ 为 null，那么 Object.prototype 是一个普通对象，也是由 new Object 得来的，那么 Object.prototype 对象的 __proto__ 属性又会指向 Object.prototype 对象自身。

这样就会形成一个无限的循环链，js 引擎在原型链上查找属性时，就会陷入死循环。

#函数的本质

在 JavaScript 中，函数本质上也是对象，是一种可调用的特殊对象。所有的函数，包括我们常用的 Object、Array 等内置函数，都是通过 new Function() 创建的：

function sum(a, b) {
  return a + b;
}

// 等价于
const sum = new Function("a", "b", "return a + b");

这就解释了为什么函数对象（比如 Object 函数）的 __proto__ 会指向 Function.prototype：

#Function 函数的特殊性

Function 函数是 JavaScript 引擎启动时就创建的特殊对象，它是所有函数的根源。作为一个特例，它的 __proto__ 指向自己的原型对象 Function.prototype：

而 Function.prototype 作为一个普通对象，是由 new Object() 创建的，所以它的 __proto__ 指向 Object.prototype，形成了完整的原型链闭环。

#原型链的实际应用

让我们看一个实际的例子，展示原型链在开发中的应用：

// 创建一个自定义数组方法
Array.prototype.sum = function() {
  return this.reduce((acc, curr) => acc + curr, 0);
};

const numbers = [1, 2, 3, 4];
console.log(numbers.sum()); // 输出：10

// 这个例子展示了如何通过原型链扩展内置对象的功能
